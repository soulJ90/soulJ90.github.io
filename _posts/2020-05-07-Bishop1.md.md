<h1 id="introduction">1 Introduction</h1>
<h3 id="history-of-pattern-recognition">History of pattern recognition</h3>
<ul>
<li>In the 16th century  <strong>Astronomical observation</strong>  :  discovering the empirical laws of planetary motion.</li>
<li>early twentieth century, <strong>Quantum physics</strong>  :  discovering of regularities in  atomic spectra.</li>
<li>In <strong>Machine learning</strong> :  automatic discovering of regularities in data through the use of computer algorithms and taking actions such as classifying the data into different categories.</li>
</ul>
<h3 id="types-of-pattern-recognition-problem">Types of pattern recognition problem</h3>
<ol>
<li>
<p>Supervised  learning</p>
<ul>
<li>The training data comprises examples of the input vectors along with their corresponding target vector.</li>
<li>ex) Classification, Regression</li>
</ul>
</li>
<li>
<p>Unsupervised learning</p>
<ul>
<li>The training data consists of a set of input vectors x without any corresponding target value.</li>
<li>ex) Clustering, Density estimation, Visualization using dimension reduction</li>
</ul>
</li>
<li>
<p>Reinforcement learning (SuttonandBarto,1998)</p>
<ul>
<li>The problem of ﬁnding suitable actions to take in a given situation in order to maximize a reward.</li>
<li>In contrast to supervised learning, the learning algorithm is not given examples of optimal outputs, but must instead discover them by a process of trial and error.</li>
<li>Typically there is a sequence of states and actions in which the learning algorithm is interacting with its environment.</li>
<li>Trade off between <em>exploration</em> and <em>exploitation</em></li>
</ul>
<blockquote>
<p><em>exploration</em> : the system tries out new kinds of actions to see how effective they are.<br>
<em>exploitation</em> : the system makes use of actions that are known to yield a high reward.</p>
</blockquote>
</li>
</ol>
<h3 id="basic-part-in-the-application-of-machine-learning">Basic part in the application of machine learning</h3>
<ul>
<li>Probability theory</li>
<li>Decision theory</li>
<li>Informatio theory</li>
</ul>

<h1 id="probability-theory">1.2 Probability Theory</h1>
<p>A key concept in the ﬁeld of pattern recognition is that of uncertainty. It arises both through noise on measurements, as well as through the ﬁnite size of data sets. Probability theory provides a consistent framework for the quantiﬁcation and manipulation of uncertainty and forms one of the central foundations for pattern recognition</p>
